1A) The weighted average entropy of a split is:


$$\text{(fraction of points in left data set)}\ * H(R^-_{j,s}) + \text{(fraction of points in right data set)} \ * H(R^+_{j,s})$$

where entropy $H(R_m)$ of data in a region $R_m$ is given by

$$H(R_m) = - \sum_k \hat{P}_{mk} \log_2 \hat{P}_{mk}.$$

The $\hat{P}_{mk}$ is the empirical probability, which is in this case the fraction of items in region m that are of class k (see lecture notes for more details).
For this dataset, we will compute the weighted average entropies for the following potential splits of the data, $f_i \ge s$ where s is the split value. The points from the data set D in the "right" side of a split are $R^+_{j,s}(D) = \{x \in D | x_j \ge s\}$, and similarly similarly the points in the left side of the split are $R^-_{j,s}(D) = \{x \in D | x_j \lt s\}$.

- Feature $f_1 \ge 1.5$
- Feature $f_1 \ge -1.5$
- Feature $f_1 \ge 0.0$
- Feature $f_2 \ge 0.0$
We will compute the entropy of the first split $f_1 \geq 1.5$ as an example:
There are two points in the right split, of which both are the same class, so $R^+_{1,1.5}(D) = -(\log_2 1) = 0$. In the left split there are 4 points, where $3/4$ of the points are positively labeled and $1/4$ are negatively labeled, so $R^-_{1,1.5} = -(\frac{3}{4} \log_2 (\frac{3}{4}) + \frac{1}{4} \log_2 (\frac{1}{4})) \approx 0.8113$. Thus, the weighted average entropy is


$$\frac{4}{6} R^-_{1,1.5}(D) + \frac{2}{6} R^+_{1,1.5}(D) \approx 0.541$$


Make sure that you use log base 2 in computing entropies.

Construct a decision tree using a greedy algorithm that picks the next best split, based on picking the split that minimizes the weighted average entropy of that split. Enter the tree as a list of nodes in left-to-right pre-order traversal order. In this representation, each node is either:

- a leaf node representing a label, either 1 (for the positive label) or -1 (for the negative label).
- a feature test followed by a No node and a Yes node, where the feature test is written as a list containing two elements: a feature index (integer 1 or 2 corresponding to either feature $f_1$ or feature $f_2$) and a split value s (floating point). So for instance, [1, 0.5] represents the test "Feature $f_1 \ge 0.5$". If $f_1 \ge 0.5$ is False, then the result of the following No node should be output, otherwise the result of the (later appearing) Yes node should be output. We will assume that the split values in these tests are always chosen to be half-way between the feature values in training data points.
Note that the left child of a feature test represents the No branch and the right represents the Yes branch. So an example of a tree (this tree doesn't work for this dataset, we're just showing you an example of the notation) is:

[[2, 0.5], 1, [1, 1.5], -1, 1]
This example tree says: if $f_2 \ge 0.5$ is False, then output label 1. (Or equivalently, if $f_2 \lt 0.5$, then output label 1.) Otherwise, if $f_1 \lt 1.5$, then output label -1, else output label 1.
Another example is:
[[2, 0.5], [1, 1.5], -1, 1, 1] In this case, this tree indicates that if $f_2 \lt 0.5$, we should return the result of the next node which in this case is another sequence of feature test, No, and Yes nodes; that second node says, if $f_1 \lt 1.5$ then return -1 otherwise 1. But if $f_2 \ge 0.5$ from the first test was Yes, we should ignore the first No branch and instead return the final leaf value 1.
